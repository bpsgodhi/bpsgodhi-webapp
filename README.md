# {{APP_NAME}} — Google Sheet backed app

Generated by the **sheet-app-builder** skill. React + Vite + Tailwind + Zustand, with a Google Sheet (via Apps Script) as the only backend.

Features: login (user/admin) + tab access control · auto-generated IDs · dashboard with charts · add / view / search / edit / delete for every module.

## Run

```bash
npm install
copy .env.example .env      # Windows  (or: cp .env.example .env)
# edit .env -> VITE_APPSCRIPT_URL + VITE_SPREADSHEET_ID
npm run dev
```

## Google Sheet setup

1. `npm run gen-db` → creates `<AppName>-Database.xlsx`. Upload it to Google Drive → open with Google Sheets → **File → Save as Google Sheets**. Copy the **Spreadsheet ID** (URL part between `/d/` and `/edit`).
2. In that sheet: **Extensions → Apps Script** → paste `google-apps-script/Code.gs` → **Deploy → New deployment → Web app** (**Execute as: Me**, **Who has access: Anyone**). Copy the `/exec` URL.
3. Put both into `.env`, then **restart** `npm run dev`.
4. Login: **admin / admin123** (change it in the Login sheet).

> If you didn't import the xlsx, run `setup()` in the Apps Script editor once to create the Login sheet + admin, and add each module's sheet/tab with the headers shown in `app.config.json`.

## Customizing

Everything is driven by [`src/app.config.json`](src/app.config.json):
- **Add a field**: add it to a module's `fields` + add the matching column header to that sheet (or re-run `npm run gen-db` for a fresh xlsx).
- **Add a whole section/tab**: add a new object to `modules` (with its own `sheet`, `fields`, `idColumn`/`idPrefix`) + create that tab in the sheet.
- **Theme color**: change `theme` for display, and find-replace `sky-` → `<color>-` across `src/`.

The Apps Script reads headers live from each sheet, so **column headers must exactly match the field `key`s** in `app.config.json`.
